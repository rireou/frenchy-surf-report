Frenchy Surf Report data and freshness

Frenchy Review publishes current surf conditions in crawlable HTML, JSON and JSON-LD from one calculation result.

How current HTML is produced

A Netlify Function runs when a surf-report page is requested. It fetches the current marine, wind and weather inputs, loads the trusted 2026 tide table, and executes the same Seaford or Middleton calculation used by the visible interface.

The function writes the calculated wave, swell, wind, tide, timestamps and status into the initial HTML response before it reaches the browser. JavaScript is not required to read the current summary.

The response may be cached by Netlify for five minutes. Every report contains issued_at and valid_until. Data older than its validity window is marked expired; if no verified result is available, it is marked unavailable. An old build-time value is never labelled current.

Calibrated and regional reports

  • Seaford: spot-specific Frenchy calculation.
  • Middleton: spot-specific Frenchy calculation.
  • Adelaide Mid Coast and South Port: clearly labelled regional Mid Coast pages using the Seaford-calibrated report. They do not claim a separate spot calibration.

Machine-readable data

  • All report endpoints
  • Seaford JSON
  • Middleton JSON
  • Adelaide Mid Coast regional JSON
  • South Port regional JSON

JSON-LD embedded in each HTML page is built from the same canonical report object as these endpoints and the visible page.
